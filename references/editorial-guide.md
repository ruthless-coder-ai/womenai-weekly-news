# 每周 AI 要闻 · 选稿标准与语言风格（完整规范）

母版（字体、配色、尺寸、页眉页脚、水印）全部锁死在 `assets/style.css` 和
`scripts/build_news_cards.py` 里，每期长得一样，只换字。写文案时不需要管版式。

## 选稿标准（三条新闻怎么挑）

- 三条覆盖**不同角度**，一条一个：
  1. **钱 / 成本 / 门槛**：价格变动、免费开放、订阅调整、算力成本，普通人钱包能感到的
  2. **人 / 工作 / 社会**：就业、教育、创作者、监管、AI 对日常生活与关系的影响
  3. **工具 / 模型**：新模型、新产品、新功能，普通人马上能上手用的
- 偏普通人能看懂、有切身感。**不选纯行业内幕**（融资轮次、估值、高管人事）。
- **读者主体是国内的中文读者（小红书发布）**，按国内读者会不会关心来选：
  - 国内大厂动态（豆包、Kimi、DeepSeek、通义、混元）、国内监管新规、
    国内可直接用上的开源模型，天然切身，优先考虑
  - 海外新闻不排斥，但门槛更高：要么影响力足够大（改变行业走向、全球都在
    讨论的事），要么波及国内用户也用得到的产品；纯美国本地的事
    （海外订阅价格微调、只影响美国职场的诉讼）不选
- 提到国内公司**不写「中国AI公司 XX」这种外媒翻译腔**——说「国产大模型」
  「国产 AI」或直接用公司中文名（月之暗面、字节、阿里、智谱）。
- **必须落在本期周一到周日范围内**，且每条有可查信源。上周的不硬塞，宁可换角度也不放旧闻。
- 每条记下：来源媒体名 + 报道日期（渲染时写进 source 字段，如 `TechCrunch · 06.24`）。

## 语言风格（非常重要，严格遵守）

这是女性写给女性看的内容。

- **标题必须带大厂名或产品名**（如 OpenAI、Gemini、豆包），读者扫一眼就知道主角是谁
- **正文客观陈述新闻事实，不人格化、不编个人经历**，不写「小美发现……」这类演绎
- 只有「对 WoMen 为什么重要」一段才用「我们」口吻，像同伴并排说话
- **不要用引号 "" 来强调**——靠断句或 `<g>薄荷绿高亮</g>`
- **不要用破折号 ——**
- **不要用「不是 A，而是 B」对仗反转句式**，读起来有 AI 腔
- 高亮克制：每张卡片挑最关键的一句或一个词标 `<g>` 即可，不要满屏绿
- AI 不用「他 / 她」指代——用「它」，或直接重复「AI」
- **写作视角是独立编辑部，不站在任何公司立场**：不用「自家」「我们的模型」
  指代任何公司的产品（反例：「对标 Fable 5，超过自家 Opus」——「自家」暴露
  写稿的是 Claude）。提到 Anthropic / OpenAI / 谷歌一律第三方口吻
- 句子可以短，不堆行业术语；专有名词第一次出现时用半句话解释

## 每张新闻卡的结构

1. `kicker`：角度标签（如 `钱 · 门槛`、`人 · 工作`、`工具 · 模型`），2–4 个字一组
2. `title`：标题，带大厂名/产品名，1–2 行，可 `<g>` 标关键词
3. `body`：正文，2–4 句客观事实；`body2` 可选，暗调补充背景
4. `why`：「对 WoMen 为什么重要」，1–2 句「我们」口吻，其中一句 `<g>` 高亮
   （小标题由脚本自动注入，不要写进 why 里）
5. `source`：`媒体名 · MM.DD`

## 封面结构

- `title`:大标题，2 行内（如 `本周 AI\n值得知道的三件事`），可 `<g>` 标关键词
- `headlines`：三条新闻各压缩成一行（≤20 字），顺序与内容卡一致
- Vol 号、日期范围、「→ 滑动看本周三条」由脚本注入或有默认值

## 常见问题速查

| 现象 | 原因 | 怎么办 |
|---|---|---|
| 中文变方块 □□□ | 渲染环境没装中文字体 | Linux 下先装 `fonts-noto-cjk`；macOS 自带 PingFang 无此问题 |
| 每期字体不一样 | 字体栈被改动 | 字体栈锁死在 style.css，不要动 |
| 脚本报 overflow 警告 | 文案太长 | 精简文案重渲染，**别改母版尺寸** |
| 文案读着有 AI 腔 | 触犯风格红线 | 对照上面的红线逐句检查重写 |
| 正文像在演一个人 | 人格化了 | 正文改回客观陈述，去掉个人经历 |
| 三条太像 | 没覆盖不同角度 | 换成钱 / 人 / 工具三个不同角度 |
